# Security Policy

## Supported versions

This repository ships a single versioned release line (currently
`1.x`). Security fixes are applied to the latest tagged release.

| Version | Supported          |
|---------|--------------------|
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a vulnerability

Please **do not open a public GitHub issue** for security
vulnerabilities. Email the corresponding author directly at
<ashrafulislamdarkeye@gmail.com> with:

* a clear description of the issue,
* a minimal reproduction (preferably a notebook cell),
* the affected version / commit hash.

We will acknowledge receipt within 5 business days and aim to ship a
fix within 30 days, coordinated with a private security advisory.

## Scope

This repository ships research code that reads the publicly available
UCI and OULAD datasets, fits tabular ML models, and writes the
artefacts to disk. The main attack-surface considerations are:

* **Untrusted pickle / joblib files.** The shipped `*.joblib` files
  were generated by the authors. Do **not** load pickle/joblib files
  from untrusted sources — pickle is unsafe by design. The notebooks
  use `joblib.load(...)` only on files written by the notebooks
  themselves in the same run.
* **Out-of-distribution student data.** The open-world rejection head
  is specifically designed to abstain on out-of-distribution inputs
  (§5.5 and §8.3 of the manuscript); it is not a substitute for input
  validation at the deployment boundary.
